## Export Gutenberg to Markdown — README

Основание: итог нашей работы над плагином, включающий все обсуждённые улучшения: проверки PHP/WP, локализацию, минификацию JS, плавные уведомления, поддержку `.po/.json` и требования wordpress.org.

## Описание

Плагин добавляет в блоковый редактор Gutenberg панель «Export to Markdown», позволяющую в один клик:

* скачать текущую запись в формате `.md`;
* скопировать Markdown в буфер обмена.

Конвертация выполняется в браузере через библиотеку Turndown. Плагин не делает внешних запросов, не сохраняет файлы на сервер и работает полностью локально.

Генерируется YAML front matter: `title`, `slug`, `date`, `categories`, `tags`.

Плагин оптимизирован под требования wordpress.org: использует `wp_set_script_translations`, имеет минимальную загрузку, поддерживает локализацию и корректно проверяет версии PHP/WP.

## Возможности

* Экспорт Markdown прямо из редактора Gutenberg.
* Два режима: загрузка `.md` и копирование Markdown.
* Автоматический front matter.
* Поддержка всех стандартных блоков Gutenberg.
* Плавные анимации уведомлений.
* Полная локализация (EN, RU, UK, DE, ES, IT, FR).
* Минифицированная версия JS (`export-md.min.js`) с автоматическим переключением по `SCRIPT_DEBUG`.
* Проверка PHP/WP и автоматическая деактивация при несоответствии требований.

## Требования

* WordPress 6.0+
* PHP 7.4+
* Активный редактор Gutenberg

Фактически код работает начиная с PHP 5.3, но официальный минимум выбран как 7.4.

## Установка

1. Скопируй папку `simple-export-md` в каталог `wp-content/plugins/`
2. Убедись, что внутри есть:

   * `simple-export-md.php`
   * `assets/export-md.js`
   * `assets/export-md.min.js`
   * `assets/turndown.js`
   * `languages/` с `.po` и `.json` файлами
3. Включи плагин в меню «Плагины».

## Использование

1. Открой любую запись/страницу в Gutenberg.
2. Включи правую панель настроек (иконка ⚙).
3. Найди секцию **Export to Markdown**.
4. Нажми:

   * **Download .md** — для скачивания файла,
   * **Copy Markdown** — для копирования в буфер обмена.

После успешного выполнения появится аккуратное уведомление с плавной анимацией.

## Структура плагина

```
simple-export-md/
  simple-export-md.php
  assets/
    export-md.js
    export-md.min.js
    turndown.js
  languages/
    simple-export-md-ru_RU.po
    simple-export-md-uk.po
    simple-export-md-de_DE.po
    simple-export-md-es_ES.po
    simple-export-md-it_IT.po
    simple-export-md-fr_FR.po
    *.json (сгенерированные wp i18n make-json)
  readme.txt
```

## Локализация

Для JS-локализации WordPress требует `.json` файлы.
Генерируются командой:

```
wp i18n make-json languages --no-purge
```

или в Windows:

```
npx @wordpress/i18n-cli make-json languages --no-purge
```

Украинский язык должен иметь код `uk` (не `uk_UA`).

## Минификация JS

Минифицированная версия собирается через `terser`:

```
npx terser assets/export-md.js -c -m -o assets/export-md.min.js --source-map
```

В плагине автоматически выбирается:

* `export-md.min.js` — в продакшене,
* `export-md.js` — если включён `SCRIPT_DEBUG`.

## Проверки версий PHP/WP

Плагин проверяет:

* PHP >= 7.4
* WordPress >= 6.0

Если требования не выполнены:

* в админке показывается уведомление,
* плагин автоматически деактивируется,
* дальнейшая загрузка прекращается.

Это предотвращает потенциальные фатальные ошибки.

## Ограничения

* Кастомные блоки экспортируются как HTML-фрагменты (как у всех Markdown-конвертеров).
* Медиафайлы не выгружаются и не упаковываются — используются абсолютные ссылки.
* Clipboard API должен быть доступен в браузере.
